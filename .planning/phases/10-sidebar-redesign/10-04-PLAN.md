# Plan 10-04: Sidebar UX Polish (Gap Closure)

<objective>
Fix 6 issues from human-verify feedback: (1) Folders tab consistency - show only generated, add folder picker, (2) spinner bug on existing tag regeneration, (3) placeholder entries for new items being generated, (4) prominent generating banner in sidebar, (5) auto-open sidebar on non-sidebar generation, (6) FolderPickerModal for new folder generation.
</objective>

<action>
**1. FolderPickerModal in modals.ts:**
Add a `FolderPickerModal extends SuggestModal<TFolder>` that:
- Shows all eligible folders (folders with at least one non-self-test .md file)
- Filters by query (folder.path.toLowerCase().includes(q))
- Calls `onSelect(folder.path)` on selection

**2. Folders tab consistency (D-18, D-19) in sidebar.ts:**
Rewrite `renderFoldersPanel` to:
- Add "Generate for new folder" button at top (opens FolderPickerModal)
- Show ONLY folders that have a `_self-test.md` (generated) - remove the "Not generated" section entirely
- Also render placeholder rows for folders in `generatingFolders` that don't have a self-test yet (D-21)

**3. Spinner bug for existing tags (D-20) in sidebar.ts:**
The issue is that `generateForTag` calls `this.refresh()` before the generatingTags Set has the key added (it gets added inside `GenerationService.generate()`). Fix by:
- In `generateForTag`, add the tag to `generationService.generatingTags` BEFORE calling refresh, then call `generate()`, then remove in finally block
- Same pattern for `generateForLinks` with `generatingLinks`
- Same pattern for `generateForFolder` with `generatingFolders`

**4. Placeholder entries for new items (D-21) in sidebar.ts:**
In each panel renderer (renderFoldersPanel, renderTagsPanel, renderLinksPanel):
- After rendering existing files, iterate over the corresponding generating Set
- For each key in the Set that doesn't already have a rendered row (no matching file), render a placeholder row with name, no date, isGenerating=true

**5. Generating banner (D-22) in sidebar.ts:**
In `renderPanel()`, after the tab bar and before the panel content:
- Check if ANY generating Set has entries (generatingFolders.size + generatingTags.size + generatingLinks.size > 0)
- If yes, render a banner div with class `active-recall-generating-banner` containing a spinner and text "Generating self-test..."
- Add CSS for the banner: background `var(--interactive-accent)` at 10% opacity, padding 8px, rounded corners, flex row with spinner + text, margin-bottom 8px

**6. Auto-open sidebar (D-23) in main.ts:**
Create an `openSidebarWithTab` helper function:
```typescript
async function openSidebarWithTab(app: App, plugin: ActiveRecallPlugin, tab: ActiveTab): Promise<void> {
    plugin.settings.activeTab = tab;
    await plugin.saveSettings();
    const activateView = buildActivateView(app);
    await activateView();
    // After sidebar is open, refresh to show the generating state
    setTimeout(() => refreshSidebarIfOpen(app), 100);
}
```

Update command palette/context menu handlers to call `openSidebarWithTab` before generation:
- `generate-self-test` (folder): `openSidebarWithTab(app, plugin, 'folders')` then generate
- `generate-self-test-by-tag`: `openSidebarWithTab(app, plugin, 'tags')` then generate
- `generate-self-test-from-links`: `openSidebarWithTab(app, plugin, 'links')` then generate
- `generate-self-test-for-note`: no sidebar tab for notes, skip auto-open
- File context menu (single note): skip auto-open
- Folder context menu: `openSidebarWithTab(app, plugin, 'folders')` then generate

**7. CSS additions in styles.css:**
```css
.active-recall-generating-banner {

  background: color-mix(in srgb, var(--interactive-accent) 12%, transparent);
  font-size: 0.85em;
  color: var(--text-normal);
}
```
